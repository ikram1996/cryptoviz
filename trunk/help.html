CryptoViz Help


CryptoViz is a visualization and learning tool for Cryptography algorithms. Cryptoviz uses DES (Data Encryption Standard) to demonstrate how cipher algorithms function.

Quick DES

Quick DES is a simple encryption utility built into CryptoViz. This allows the user to test encryptions in other areas of the program.

Enter any amount of text into the first text area, in either ASCII or hex.

Alternatively, browse for a text file that you would like to encrypt.

Enter your preferred key. This should be 8 hexadecimal characters.

Press the Encrypt button to encrypt the data. The first block of input plain text is converted to cypher text and shown in the output box.


Key Visualization

The Key Visualization page shows the Data Encryption Standard algorithm for producting 16 subkeys from one key.

Enter an 8 character key in plain ASCII text. (If key is longer than 8 characters, it will be truncated.)

Press the "To Binary" button. CryptoViz converts your ASCII text into it's binary equivalent, and displays this in the appropriate text box.

Also, a graphic representation of the binary key appears at the top of the page.

The following buttons are functions used by DES to permute the key and generate subkeys.

The user can click on each of these buttons to see how the bits in the binary key are permuted.

The PC2 buttons generate each of the 16 subkeys.

The slider allows the user to make the visualization faster or slower.

The Do All button performs each function in order and displays all 16 subkeys.


DES Visualization

The DES Visualization page shows the entire process of encrypting one block of data using the DES encryption algorithm.

The Fiestel Structure is displayed, with blank areas for each data area.

In the Plain Text block, enter 64 bits of data, in either ASCII or Hex.

In the Key block, enter 64 bits of data, in either ASCII or Hex.

Next, click Generate Keys to generate all of the 16 subkeys. This process is not visualized in this window, but can be seen in the "Key Visualization" window.

Next, click the Encrypt button. This fills out the entire Fiestel structure.

All data is converted to hexadecimal.

The user can also Visualize each round of the Fiestel structure, by click the Visualize Round # button.

This brings up a new window that shows the round in more detail.

This shows the incoming binary string at the top of the window. It is then split in to Left and Right halves.

The E button expands the right half of input. This is visualized at the bottom of the window.

The X button exclusively ORs this with the right half of input.

The S button is for S Box permutation.

The P button is for the next permutation.

Finally, this is exclusively ORed with the left half of input. The result of this becomes the final right half of output. The left half of output is the original right half. This is shown by the lines connecting the text boxes.
